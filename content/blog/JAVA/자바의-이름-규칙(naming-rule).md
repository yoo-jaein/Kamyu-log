## 이름 규칙

이름 규칙(Naming Rule)이란 클래스, 메소드, 변수, 상수와 같이 식별자의 이름을 지을때 따라야 하는 규칙이다.

## 클래스의 이름 규칙

클래스의 이름 규칙은 Camel Case 모델이라고 한다. 단어의 모습이 낙타 등과 같이 울룩불룩하다고 하여 붙여진 이름이다.

- 클래스 이름의 첫 문자는 대문자로 시작한다.
- 둘 이상의 단어가 묶여서 하나의 이름을 이룰 때, 새로 시작하는 단어는 대문자로 한다.

```java
class PineApple {...}
class Pine {...}
class Apple {...}
```

## 메소드와 변수의 이름 규칙

클래스의 이름과 구분을 하기 위해 메소드와 변수는 변형된 Camel Case 모델을 사용한다. 

- 메소드와 변수 이름의 첫 문자를 소문자로 시작한다.

```java
PineApple pineApplePizza;
```

## 상수의 이름 규칙

변수의 이름과 구분이 되도록 상수는 모든 문자를 대문자로 구성하는 것이 관례이다.

```java
final int MAX_SIZE = 1000000;
```

## 마무리
1. 클래스는 `Camel Case 모델`을 따라 첫 문자가 대문자이다.
2. 메소드와 변수는 `변형된 Camel Case 모델`을 따라 첫 문자는 소문자이다.
3. 상수는 변수와 구분되도록 모든 문자를 대문자로 구성한다.
