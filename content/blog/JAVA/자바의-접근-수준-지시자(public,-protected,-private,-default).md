## 접근 수준 지시자

자바의 접근 수준 지시자(Access-level modifiers)란 어떤 클래스/변수/메소드에 대해 __외부에서 접근할 수 있는 권한을 설정 해주는 키워드__를 말한다. 접근 수준 지시자들의 종류는 다음과 같다.

- 클래스 정의 대상 : public, default
- 인스턴스 변수와 메소드 대상 : public, protected, private, default

## public
> '공공의' 라는 뜻과 같이 모두에게 열려있는 클래스/변수/메소드에 붙인다.  

- 클래스를 정의할 때
    - __위치에 상관없이 어디서든 해당 클래스의 인스턴스를 생성할 수 있다.__  
</br>

- 인스턴스 변수와 메소드를 대상으로 할 때
    - __어디서든 접근이 가능하다.__  


## default
> 가장 평범한 상태. 동일한 패키지에서 접근할 수 있다.  

- 클래스를 정의할 때
    - __동일 패키지__로 묶인 클래스 내에서만 인스턴스 생성이 가능하다.
    - 'default'라는 키워드를 사용하는 것이 아니라 __'아무 선언도 하지 않은 상태'를 의미한다.__

```java
class Duck {...} // default 클래스
```

- 인스턴스 변수와 메소드를 대상으로 할 때
    - __동일 패키지__로 묶인 클래스 내에서만 접근이 가능하다.


## protected
> 상속 관계에 있어서 믿을만한 클래스들만 해당 변수/메소드에 접근을 허용한다.  

- 인스턴스 변수와 메소드를 대상으로 할 때
    - __default 선언이 허용하는 접근__을 모두 허용하며, __상속 관계__에 있는 다른 클래스에서도 접근이 가능하다.



## private
> '개인의' 라는 뜻처럼 해당 변수/메소드를 소유한 클래스 내에서만 접근을 허용한다.  

- 인스턴스 변수와 메소드를 대상으로 할 때
    - __선언된 클래스 내에서만 접근이 가능하다.__


## 마무리
1. `public`한 클래스/변수/메소드는 `어디에서든지` 접근 가능하다.
2. `default`한 클래스/변수/메소드는 `동일 패키지에서` 접근 가능하다.
3. `protected`한 변수/메소드는 `default 범위&상속 관계에서` 접근 가능하다.
4. `private`한 변수/메소드는 `선언된 클래스 내에서만` 접근 가능하다.
