## 객체지향 언어

- 자바는 객체지향 언어로써 클래스를 중심으로 코드를 작성한다. __객체지향 프로그래밍 언어는 현실 세계의 개체(Entity)를 하나의 객체로 만들어 소프트웨어를 개발할 때 객체들을 조립해서 프로그램을 작성할 수 있도록 하는 언어이다.__
</br></br>

- 객체지향 언어의 특징은 다음과 같다.
    - 상속을 통한 __재사용과 시스템의 확장__이 용이하다.
    - 코드의 __재활용성__이 높다.
    - 대형 프로그램의 작성이 용이하다.
    - 소프트웨어 개발 및 __유지보수__가 용이하다.
</br></br>

- 객체지향 언어의 구성 요소는 다음과 같다.
    - __`객체(Object)`: 데이터와 메소드를 결합시킨 실체.__
    - __`클래스(Class)`: 하나의 공통된 특성을 표현하는 요소.__
    - `메시지(Message)`: 객체들 간의 상호작용에 사용되는 수단.
</br></br>

-> 여기서 __객체__와 __클래스__에 대해서 더 자세히 알아보자.

## 객체의 정의

- __객체(Object)란 데이터와 데이터를 처리하기 위한 메소드를 결합시킨 실체이다.__ 데이터 구조와 그 위에서 수행되는 연산들을 가지고 있는 소프트웨어 모듈이라고도 할 수 있다. 

## 클래스의 정의

- 자바 프로그램의 소스코드를 살펴보면 데이터와 기능이 있음을 알 수 있다. 데이터는 변수, 상수와 같이 프로그램상에서 유지하고 관리해야 할 데이터이며, 기능은 그 데이터를 처리하는 것이다. __클래스는 데이터와 메소드 이 둘을 포괄하는 단어이다.__

```java
public class MyClass {
    int x = 5;

    public static void main(String[] args) {
        MyClass myObj = new MyClass();
        System.out.println(myObj.x);
    }

    public static int add(int num) {
        x += num;
        return x;
    }
}
```

- 위 예제에서 `변수 x`는 프로그램상의 `'데이터'`이며, `메소드 add`는 프로그램상의 `'기능'`이다. __변수 x와 메소드 add는 긴밀히 연관되어 있고 이 둘을 묶는 것이 `클래스` MyClass__이다. 이렇게 연관있는 데이터와 메소드를 묶어 주는 것이 '클래스'의 역할이다.  
</br>

- 이제 클래스를 정의했으니 이를 활용해야 한다. 클래스는 붕어빵 틀과 같아서 빵을 만들어 낼 수 있지만 그 자체를 사용할 수는 없다. 따라서 이 틀을 이용하여 `'인스턴스'`라는 붕어빵을 찍어내야 한다.

## 인스턴스 변수와 메소드

- 먼저 클래스 내부의 변수와 메소드의 명칭을 알아보고 가자. 위에서 정의한 클래스 MyClass를 간략하게 표기해보았다.

```java
public class MyClass {
    // 인스턴스 변수
    int x = 5;

    // 인스턴스 메소드
    public static int add(int num) {...}
}
```

- 위와 같이 클래스 내에 위치한 변수와 메소드를 가리켜 각각 `인스턴스 변수`(멤버 변수 또는 필드), `인스턴스 메소드`라고 부른다. 
</br></br>

- 인스턴스 변수는 같은 클래스 내에 위치한 메소드 내에서 접근이 가능하다.   


## 인스턴스의 생성

- 이제 인스턴스를 찍어내려고 한다. 이것을 __클래스의 인스턴스화(Instantiation)라고 하며, 키워드 `new`를 사용한다.__

```java
new myClass(); // 클래스 myClass의 인스턴스화
```

- 위의 문장을 실행하면 __클래스 myClass에 정의된 변수와 메소드를 담고 있는 '인스턴스'가 생성되고 실제 메모리 공간에 존재하게 된다.__  
</br>

- 그런데 이 상태로는 만들어진 인스턴스에 접근할 수 없다. 따라서 __인스턴스를 참조하기 위한 새로운 변수가 필요하다. 이것을 `'참조변수'`라고 한다.__


```java
MyClass myObj; // 참조변수 myObj 선언

// 참조변수 myObj가 새로 생성되는 인스턴스를 가리킴
myObj = new MyClass(); 
```

> 참고! 클래스를 기반으로 만들어진 인스턴스를 가리켜 객체라고도 한다.

## 참조변수
- 참조변수(Reference Variable)란 키워드 `new`를 통해 생성된 새로운 인스턴스를 가리키는 변수이다. 

```java
MyClass myObj1 = new MyClass();
MyClass myObj2 = new MyClass();
```

- 참조변수의 특징은 다음과 같다.
    - 참조하는 인스턴스를 바꿀 수 있다.
    - 둘 이상의 참조변수가 하나의 인스턴스를 참조할 수 있다.
    - null을 대입하면 아무런 인스턴스도 참조하지 않도록 할 수 있다.

## 마무리

1. 클래스(Class)란 두 개 이상의 유사한 객체들을 묶어서 하나의 공통된 특성을 표현하는 요소이다.
2. 클래스를 통해 인스턴스를 찍어낼 수 있다.
3. 참조변수는 인스턴스를 가리키는 변수이다.
