## 인스턴스 변수와 메소드

지난 글에서 인스턴스 변수에 대해 정리했다. 

```java
public class MyClass {
    // 인스턴스 변수
    int x = 5;

    // 인스턴스 메소드
    public static int add(int num) {...}
}
```

- 위와 같이 클래스 내에 위치한 변수와 메소드를 가리켜 각각 '인스턴스 변수'(멤버 변수 또는 필드), '인스턴스 메소드'라고 부른다. 


## 클래스 변수

- 만약 변수 앞에 `static`이 붙어 있다면 해당 변수는 __인스턴스 변수가 아니라 `클래스 변수`가 된다.__

```java
class Request {
    static int count = 0;

    Request() { // 생성자
        count++;
        System.out.println(count+"개의 Request가 생성되었습니다.");
    }
}
class Main {
    public static void main(String[] args) {
        Request r1 = new Request();
        Request r2 = new Request();
    }
}
```

- 클래스 변수의 특징은 다음과 같다.
    - __변수가 선언된 클래스의 모든 인스턴스들이 `공유`한다.__
    - 어떤 __인스턴스에도 속하지 않은 상태로, 메모리 공간에 `딱 하나`만 존재한다.__
    - __`인스턴스 생성 이전에` 이미 메모리에 존재한다.__ 따라서 __인스턴스 생성 이전에 클래스 변수에 접근이 가능하다.__
    - 이 변수가 선언된 클래스의 인스턴스들은 이 변수에 바로 접근할 수 있다.
    - 접근 수준 지시자(public, default, protected, private)의 규칙을 적용받는다.  
</br>

- 그래서 클래스 변수는 모든 인스턴스에서 참조해야 하는 값이나 생성된 인스턴스의 수를 관리할 때 유용하다.

## 클래스 메소드

- 클래스 내에 정의된 메소드에 `static` 선언을 붙이면 `클래스 메소드`가 된다.

```java
class Main {
    public static void main(String[] args) {
        Request r1 = new Request();
        Request r2 = new Request();
    }
}
```

- 클래스 메소드의 특징은 클래스 메소드의 것과 유사하다.
    - `인스턴스 생성 이전`부터 호출이 가능하다.
    - 어떤 인스턴스에도 속하지 않는다.
    > 그렇기 때문에 인스턴스 변수에 접근할 수 없고 인스턴스 메소드를 호출할 수 없다.


## 자바 프로그램의 main 메소드

자바 프로그램의 main 메소드는 반드시 다음과 같은 모습으로 선언되어야 한다.

```java
public static void main(String[] args) {
    ...
}
```

- public static으로 선언해야 하는 이유는 다음과 같다.
    1. 클래스 외부에서 실행된다. (public)
    2. 인스턴스가 생성되기 이전에 호출된다. (static)

## 마무리

1. `static`이 붙은 변수와 메소드는 `클래스 변수`와 `클래스 메소드`이다.
2. 클래스 변수와 메소드는 `인스턴스 생성 이전에` 이미 메모리에 존재한다. 따라서 인스턴스 생성 이전부터 호출이 가능하다.
3. 클래스 변수와 메소드는 해당 클래스의 `모든 인스턴스가 공유`한다.
